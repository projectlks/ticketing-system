၁။ စနစ်အကြောင်း မိတ်ဆက်

ဤ Ticketing System သည် အဖွဲ့အစည်းအတွင်း ဖြစ်ပေါ်လာသော ပြဿနာများ (Incidents)၊ လိုအပ်ချက်များ (Service Requests) ကို စနစ်တကျ မှတ်တမ်းတင် ဖြေရှင်းနိုင်ရန် ခေတ်မီနည်းပညာများ အသုံးပြု၍ တည်ဆောက်ထားသော ခိုင်မာသည့် Helpdesk application တစ်ခု ဖြစ်ပါသည်။

Modern Stack

  • Next.js ဖြင့် မြန်ဆန်သော UI.
  • Node.js API endpoints.
  • Tailwind CSS responsive design.

Robust Infrastructure

  • PostgreSQL 17 (Primary-Replica).
  • Prisma ORM for database access.
  • Redis caching & socket storage.

၂။ အသုံးပြုသူ အဆင့်များ (Roles)

Role အဆင့် (၄) ဆင့်ခွဲခြားထားပြီး တာဝန်နှင့် လုပ်ပိုင်ခွင့်များကို အောက်ပါအတိုင်း ရှင်းလင်းစွာ ခွဲဝေထားပါသည်။

LEVEL_1

Standard User / Requester

ပုံမှန် ဝန်ထမ်းများ သို့မဟုတ် ပြဿနာကို စတင်တင်ပြသူများ။

  • Ticket အသစ်များ ဖန်တီးနိုင်သည်
  • မိမိ Ticket အခြေအနေ ကြည့်နိုင်သည်
  • Comment ရေးသားနိုင်သည်
LEVEL_2

Support Agent

ဌာနဆိုင်ရာ တာဝန်ခံများ၊ ပြဿနာကို လက်ခံဖြေရှင်းပေးရမည့် Support Team။

  • ဌာနတွင်း Ticket အားလုံးကို မြင်နိုင်သည်
  • Ticket ကို Assign ယူနိုင်သည်
  • Status အခြေအနေ ပြောင်းလဲနိုင်သည်
LEVEL_3

Specialist Engineer

အဆင့်မြင့် နည်းပညာကျွမ်းကျင်သူများ (Escalation Team)။

  • Escalate လုပ်ထားသော Ticket များကိုင်တွယ်သည်
  • ဝန်ထမ်းအချင်းချင်း Re-assign လုပ်နိုင်သည်
  • Critical SLA များကို အဓိကတာဝန်ယူသည်
SUPER_ADMIN

System Administrator

စနစ်တစ်ခုလုံးကို အပြည့်အဝ စီမံခန့်ခွဲခွင့်ရှိသူ (Full Access)။

  • User Accounts စီမံခန့်ခွဲနိုင်သည်
  • Departments/Categories အသစ်ဖန်တီးနိုင်သည်
  • SLA Rules များကို သတ်မှတ်နိုင်သည်
  • Audit/Reports အားလုံးကို ကြည့်နိုင်သည်

Status ရှင်းလင်းချက်

NEW

Ticket အသစ်စတင်ဖွင့်ထားပြီး မည်သူမှ စတင်ကိုင်တွယ်ခြင်း မရှိသေးသောအခြေအနေ။

OPEN

Support အဖွဲ့မှ Ticket ကိုမြင်တွေ့ပြီး Assign လူချထားပြီးဖြစ်သော အခြေအနေ။

IN_PROGRESS

သက်ဆိုင်ရာ Engineer မှ လက်တွေ့ စတင်ဖြေရှင်း ဆောင်ရွက်နေဆဲ အခြေအနေ။

RESOLVED

ပြဿနာကို ဖြေရှင်းပြီးစီးကြောင်း Support ဘက်မှ အတည်ပြုထားသော (User လက်ခံရန်စောင့်နေသော) အခြေအနေ။

ဦးစားပေး အဆင့်များ

REQUEST

P4 - Routine

သာမန် ဝန်ဆောင်မှု တောင်းဆိုချက်များ (ဥပမာ- ဆော့ဖ်ဝဲလ်သွင်းခြင်း)။ အချိန်ယူ ဖြေရှင်းနိုင်သည်။

MINOR

P3 - Low Impact

တစ်ဦးတစ်ယောက်တည်းကိုသာ သက်ရောက်သော အသေးစား ပြဿနာများ (ဥပမာ- Printer error)။

MAJOR

P2 - High Impact

ဌာနတစ်ခုလုံး သို့မဟုတ် လူအများစုကို သက်ရောက်သော ပြဿနာများ (ဥပမာ- Network Slow ခြင်း)။

CRITICAL

P1 - Emergency

လုပ်ငန်းလည်ပတ်မှု ရပ်တန့်သွားစေသော Core Server down ခြင်း။ အရေးပေါ် ချက်ချင်းဖြေရှင်းရမည်။

၄။ အဓိက လုပ်ဆောင်ချက်များ (Features)

Automated Ticket Creation

System Monitoring ပြုလုပ်နေသော Zabbix Alert များမှတစ်ဆင့် Manual ကြားခံစရာမလိုဘဲ အလိုအလျောက် Ticket ဖွင့်ပေးသည့်စနစ် ပါဝင်သည်။

SLA Tracking & Violation

Ticket အမျိုးအစားအလိုက် Response & Resolution Time များကို အချိန်တိုင်းတာပေးပြီး သတ်မှတ်ချိန်ထက် ကျော်လွန်ပါက Alert ပြပေးသည်။

Detailed Audit Trails

Changes မှန်သမျှ (Status ပြောင်းခြင်း၊ Assign ယူခြင်း) ကို ဘယ်အချိန်မှာ ဘယ်သူပြုလုပ်သွားသလဲဆိုတာကို History JSON ဖြင့် အတိအကျသိမ်းဆည်းထားသည်။

Real-time WebSocket

socket.io ကိုအသုံးပြုထားသဖြင့် User ဘက်မှ Refresh မလုပ်လုပ်စရာမလိုဘဲ အချက်အလက်နှင့် Comment အသစ်များကို ချက်ချင်းမြင်တွေ့ရမည်။

၅။ Zabbix Integration Process

စနစ်သည် IT Infrastructure ကို စောင့်ကြည့်သော Zabbix Monitoring Server နှင့် API နည်းပညာဖြင့် တိုက်ရိုက်ချိတ်ဆက်ထားပါသည်။
